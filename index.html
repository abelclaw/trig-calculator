1 What is a Function?

Imagine a machine. You put a number in one side, something happens inside, and a number comes out the other side. That's a function.

A function is a rule that takes each input and gives back exactly one output. No surprises, no randomness — the same input always gives the same output.

We write functions like this: f(x) = 2x. This means "the function f takes a number x and gives back 2 times x." The x is the input, and f(x) is the output.

Seeing a function as a graph

Every function can be drawn as a picture. The input (x) goes along the bottom, and the output goes up the side. Each point on the curve says "when the input is here, the output is there."

Move your mouse across the graph
On a graph, a function means: every vertical line crosses the curve at most once. If a vertical line hits the curve twice, the graph is not a function (because one input would give two outputs).

If f(x) = 2x, then the inverse is f¹(y) = y / 2. The inverse undoes what the original function did.

Seeing it on the graph

The inverse function is the original function reflected across the diagonal line y = x. If the point (3, 6) is on the original function, then (6, 3) is on the inverse.

Move your mouse to trace both functions
f(f¹(x)) = x and f¹(f(x)) = x. The function and its inverse cancel each other out, like how addition cancels subtraction, or how multiplying cancels dividing.

3 The Problem: Some Functions Can't Be Reversed Simply

Here's the tricky part. Consider the function f(x) = x².

If I tell you the output is 9, what was the input? Well... it could have been 3 (because 3² = 9), but it also could have been -3 (because (-3)² = 9). Two different inputs gave the same output!

A function that sends two different inputs to the same output doesn't have an inverse (at least, not in the normal sense) because you can't un-do it without ambiguity.

The solution: Restrict the domain

We fix this by choosing to only look at part of the function — a section where each output only happens once. For x², if we only look at x ≥ 0, then each output has exactly one input, and the inverse is the square root: √y.

Click to toggle domain restriction
To create an inverse, we sometimes need to restrict the domain — only use part of the function where each output happens exactly once. The square root function is the inverse of x², but only for x ≥ 0.

4 Inverse Trig Functions (like arccos)

The cosine function has exactly this problem. Many different angles give the same cosine value.

For example, cos(60°) = 0.5, but also cos(300°) = 0.5, and cos(420°) = 0.5, and infinitely more. So if someone says "the cosine is 0.5, what's the angle?" — there are infinitely many correct answers!

arccos: Picking one answer

We solve this the same way as with x²: we restrict the domain. For cosine, we only look at angles from 0° to 180° (0 to π radians). In that range, each cosine value happens exactly once.

The inverse cosine, written arccos or cos¹, answers the question: "What angle between 0° and 180° has this cosine value?"

arccos(0.5) = 60° because 60° is the angle between 0° and 180° whose cosine is 0.5. There are other angles with cosine 0.5 (like 300°), but arccos always picks the one in its restricted range.

Inverse trig functions answer the question: "What angle gives this value?" They always pick one specific answer from a restricted range, even though many angles would work. That's the whole trick!
